## 一、设计理念

**核心风格**：简约清晰、白蓝主题、Dify 风格

| 特性         | 描述                           |
| ------------ | ------------------------------ |
| **简洁纯净** | 大量留白，减少视觉噪音         |
| **清晰层级** | 明确的视觉层级，信息层次分明   |
| **专业感**   | 白色为主，蓝色点缀，企业级品质 |
| **现代化**   | 扁平化设计，卡片式布局         |
| **微交互**   | 细腻的过渡动画，提升体验       |

**设计参考**：Dify, Linear, Vercel

**页面元素**：

- **粒子动画背景**：科技感十足的粒子连线动画，体现"连接"与"智能"
- **Logo 区域**：
  - 动态发光的同心圆图标，中心带勾选标记
  - 渐变色品牌名称 "SmartLink"
- **标语区域**：
  - 主标语：智能企业级 Agent 应用管理平台
  - 副标语：自然语言驱动 · 智能编排 · 动态渲染

  - **开始探索** (主按钮)：渐变蓝底色，圆角胶囊形，跳转到探索中心
  - **体验 OpenClaw** (次级按钮)：透明底色 + 蓝色边框，打开外部链接

**设计理念体现**：

| 元素         | 体现理念                   |

| 渐变色彩     | 现代化、科技感             |
| 简洁布局     | 简约清晰的设计风格         |
| 主次按钮     | 引导用户行为，突出核心功能 |

#### 4.2.1.6 动态组件渲染数据结构

**消息数据模型**：

```typescript
interface ChatMessage {
  id: string
  role: 'user' | 'assistant'
  timestamp: number

  // 文本内容
  content?: string

  // 动态组件内容
  components?: ComponentNode[]

  // 组件交互状态
  interactionState?: Record<string, any>

  // 事件回调
  events?: MessageEvent[]
}

interface MessageEvent {
  type: 'form_submit' | 'button_click' | 'selection_change'
  componentId: string
  data: Record<string, any>
}
```

**组件渲染流程**：

```
AI响应 → 解析消息 → 识别消息类型
                         ↓
         ┌───────────────┴───────────────┐
         ↓                               ↓
    纯文本消息                      组件消息
         ↓                               ↓
    Markdown渲染              ComponentNode[]
         ↓                               ↓
         └───────────────┬───────────────┘
                         ↓
                   渲染到界面
                         ↓
                   用户交互事件
                         ↓
                   回传给AI处理
```

**交互特性**：

- 左侧面板：新建对话、搜索、对话列表、历史记录
- 中间内容区：模板选择（首次）、消息列表
- 消息布局：AI居左，用户居右
- 支持 Markdown 渲染和代码高亮
- 流式输出 AI 回复
- 动态组件渲染（基础展示、表单交互、图表、业务组件）
- 组件支持事件交互（表单提交、按钮点击等）
- 消息支持复制/重新生成
- 附件上传支持

### 4.3 智能体管理模块

智能体管理模块是系统的核心功能，基于 Role（角色）的数字员工智能体管理。包含两个子模块：

- **智能体列表**：展示已发布智能体的运行态管理

**智能体核心要素**（基于架构设计）：

| **展示页面**   | 智能体的前端展示界面       | 页面编排、组件配置                   |

### 4.5 工具管理模块

#### 概述

工具管理模块包含三个核心子模块：

- **MCP 服务器管理**：配置和管理 Model Context Protocol 服务器连接
- **Skills 管理**：管理 AI Agent 的可复用技能模块
- **LLM 模型管理**：配置和管理多模型提供商及路由策略

**设计原则**：

- **卡片式主页面**：直观展示关键信息，快速概览状态
- **下钻详情页**：分结构展示完整配置和监控数据
- **状态可视化**：通过颜色、图标清晰传达连接/运行状态
- **操作便捷**：常用操作（测试、编辑、删除）一键可达

**卡片信息字段**：

| 字段       | 描述                            | 显示规则                           |
| ---------- | ------------------------------- | ---------------------------------- |
| 状态指示器 | 🟢 已连接 / 🟡 连接中 / 🔴 断开 | 根据连接状态动态变化               |
| 服务器名称 | MCP 服务器的 name 或 title      | 截断显示，最长 30 字符             |

| 资源数     | 可用的 resources 数量           | 带📁图标                           |
| Prompt 数  | 可用的 prompts 数量             | 带💬图标，为 0 时不显示            |
| 最后活动   | 最后成功通信时间                | 相对时间格式（X 分钟前）           |

| 字段       | 描述                            | 显示规则                     |

| 版本号     | 语义化版本                      | 遵循 semver 规范             |
| 作者       | 负责团队或个人                  | 显示团队名称                 |
| 调用次数   | 最近 30 天调用总数              | 格式化显示 (12K, 1.2M)       |
| 成功率     | 执行成功率百分比                | <95% 显示橙色，<90% 显示红色 |
| 平均耗时   | 平均执行时间                    | 秒级单位，保留 1 位小数      |
| 风险等级   | 低/中/高风险                    | 高风险显示"需要审批"标注     |

| 字段       | 描述                                          | 显示规则                     |

## 六、实施建议

### 6.1 优先级建议

**第一阶段**（基础框架）：

1. 侧边栏导航组件实现
2. 欢迎页面
3. 应用列表页面

**第二阶段**（核心功能）：

1. 对话页面
2. 应用设计页面（含可视化编排功能）

**第三阶段**（资源与工具）：

1. 组件管理页面
2. MCP 管理页面
3. Skills 管理页面
4. 模型管理页面

**第四阶段**（完善优化）：

1. 系统设置（外观设置、模型提供商）
2. 数据模型管理
3. 性能优化和细节打磨

### 6.2 响应式设计

- 桌面端：完整布局（≥1440px）
- 平板端：可收起侧边栏（768px-1439px）
- 移动端：底部导航 + 全屏内容（<768px）
